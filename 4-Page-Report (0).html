In Figure 1, there is a trend of cost increasing with sqft. This is a typical linear regression problem. In more detail, this is a univariate linear regression model and we are trying to fit a hypothesis.

Our aim is to find the line of best fit for Figure 1. A line of best fit is also called a hypothesis. We can represent this hypothesis, by determining the slope and y-intercept. The intercept and slopes are weights w0 and w1 respectively. These weights are the values that stabilize the hyp

  1. In §1.1, we use sklearns LinearRegression library to determine the weights.
  2. In §1.2, we use an analytic approach. These weights will be compared to the weights determined by the sklearn library.
  3. In §1.3, we determine a cost function and apply a gradient descent learning algorithm on the cost function.

The weights from the second approach will be compared to the weights determined by an analytic solution. We compare it this manner because there are less moving parts in an analytic approach. You just need to ensure the design matrix and target are sized appropriately, and apply a series of basic linear algebra techniques such as inversion, multiplication on the design matrix and the target.

§1.2 An Analytic Approach¶

If we wanted to do this at a lower level, there are two ways to solve this. After determining the weights from with the linear regression machine learning model provided by the sklearn package, we will find the weights 'by hand' as well. There are many ways to do this. In this project, we will be employing the following two approaches to find the weights, similar to Lab 2.

Its very convenient to be able to calculate the weights with the following approach. In fact this is the best way to solve this problem for this situation (excluding sklearn)

As mentioned above, since there's only one feature to consider (sqft), we can solve for the weights with an analytic approach. Essentially, this means using basic linear algebra techniques, to solve for our weight, weights being our slope and intercept in this single-featured example.

The advantages of an analytic approach:

  • no need for gradient descent
  • no need to perform thousands of iterations
  • no need for learning rate guesswork
  • no need for feature scaling

In fact, the entire solution can be solved with an analytic approach. If the code is vectorized, an analytic approach would work up to even 10,000 features.

    return np.linalg.inv(X.T @ X) @ X.T @ y

§1.5 Gradient Descent¶

As can be seen in our function definition, we guess the weights as w0=0 and w1= 0.

With gradient descent, our objective is to minimize the weights w0 and w1 such that the cost is the lowest. Essentially, the algorithm is paying a 'cost' for the parameters it picks, so it 'learns' by picking the weights with the lowest cost.

Basically you are instructing the algorthm, 'Find me the values of w0 and w1 that gives me the minimum of a cost function.'

Recap that from the normal equation, what we're supposed to be getting

w0=1.84 and w1=2.29

def gradient_descent(X, y, learn_rate=0.00004, num_steps= 10000):
    w = np.array([0, 0]) # so instead of guessing x, we're guessing w
    for i in range(num_steps):
        grad = (X.T @ X) @ w - X.T @ y  # so.. this is a very useful equation
        w = w - learn_rate*grad
    return w

Up to now, we have only considered a single feature, the size of the house (sqft). We are limited to only a single feature if we want to be able to plot. If we had more than one feature, we cannot visualize the cost function and gradient descent actually descending. This is important because a visualizing this greatly helps understand the mechanics of gradient descent for other machine learning techniques.
